# /ctx — 진행 상황 요약

호출되면 **이 세션에서 한 작업과 현재 상태**를 다음 형식으로 정리해서 출력한다. 답변은 한국어. 텔레그램 세션이면 mcp__plugin_telegram_telegram__reply 로, 아니면 일반 텍스트로.

## 출력 형식

```
**현재 상황 (YYYY-MM-DD HH:MM)**

✅ **완료한 것 (이 세션)**
- 핵심 결과 위주로 3~7개. 코드 수정/빌드/배포/메시지 전송 등 의미있는 것만.

⏳ **진행 중 / 대기 중**
- 백그라운드 작업, 외부 처리 대기, 사람 응답 대기 같은 것들.

🚧 **블로커 (있다면)**
- 내가 못 진행하는 이유. 사람이 해야 할 액션, 빠진 권한, 결정 필요한 것 등.

🎯 **다음 액션**
- 강대종님이 멈추라고 안 하면 내가 바로 할 다음 한 가지.

📝 **할일 (~/todo/todos.md)**
- 진행중 섹션의 항목명만 짧게 나열. 상세 내용은 생략.
```

## 작성 가이드

- **이 세션에서 한 일만** 적는다. 옛날 일이나 git history 같은 건 빼고.
- **결과 중심**으로. "X 했어요" 가 아니라 "X 완료, 결과는 Y".
- **숫자/ID/경로**는 정확하게. "빌드 7번 (1.0.0+7)", "/tmp/...", "PID 7904" 같이.
- **추측 금지**. 모르는 건 "확인 필요"로 표시.
- **스크롤 안 해도 한 화면**에 들어가게. 필요시 항목을 줄이거나 짧게.
- 강대종님이 "ctx" 한 단어만 쳐도 호출되도록 짧고 명확한 트리거.

## 호출 패턴 예시

- "ctx" → 위 형식 그대로
- "ctx 짧게" → 위 형식의 절반 분량으로
- "ctx 메모요" → 메모요 관련 작업만 필터링해서 출력
