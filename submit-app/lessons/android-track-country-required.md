# Closed Testing 트랙도 국가/지역 1개 이상 필수

## 증상
release/review 페이지에서 "오류이 트랙에 선택된 국가 또는 지역이 없습니다. 이 버전을 출시하려면 국가 또는 지역을 1개 이상 추가하세요." 메시지로 저장 버튼 disabled. AAB 업로드, 출시 노트 입력, 모든 9개 콘텐츠 선언 완료된 상태에서도 막힘.

## 원인
프로덕션 트랙뿐 아니라 **Closed Testing (Alpha) 트랙도** 최소 1개 국가/지역을 명시해야 빌드를 출시할 수 있음. 신규 앱은 트랙 생성 시 국가가 0 으로 초기화돼있고, 워크시트에서 자주 빠뜨리는 항목.

## 해결
1. /tracks/{TRACK_ID}?tab=countryAvailability 진입
2. "국가/지역 추가" 버튼 클릭
3. 검색에 **"대한민국"** (Korea 가 아니라 정식명) 입력 → 체크 → 저장
4. 다이얼로그에서 "나중에" 또는 "개요로 이동" 선택
5. 다시 review 페이지로 돌아가서 저장 → 게시 개요에서 검토 전송

- [ ] /create-play-app 신규 앱 등록 자동화에 "기본 국가 = 대한민국" 자동 셋업 추가
